あなたは経験豊富なフルスタックエンジニアです。
以下の要件を満たす「最小構成のWebアプリ」を設計・実装してください。

# 目的
Jiraに自分がアサインされているチケットを自動取得し、
それをタスク一覧として表示し、
ユーザーがポモドーロタイマーで作業時間を計測すると、
その時間が対応するJiraチケットのActual Effort（Worklog）に自動反映されるアプリを作りたい。

このアプリは社内の有志活動報告用デモが目的であり、
商用レベルの厳密性やセキュリティは不要。
「ちゃんと連携している」ことが分かることを最優先とする。

# 機能要件
1. Jira REST API を使い、以下条件のチケットを取得
   - assignee = currentUser()
   - statusCategory != Done
2. チケット一覧を画面に表示
   - issue key
   - summary
   - status
3. 各チケットに「Start Timer」ボタンを表示
4. タイマー開始時
   - フロント側で計測開始（25分固定 or 手動Stop）
   - Jiraには何も送信しない
5. タイマー停止時
   - 経過時間（秒）を計算
   - Jira Worklog API を使って工数を登録
6. 任意で以下も実装
   - タイマー開始時に Jira ステータスを「In Progress」に変更
   - タイマー停止時に元のステータスへ戻す、または変更しない

# 非機能要件
- シングルユーザー前提
- 認証は Jira API Token（Basic Auth）
- データ永続化は不要（メモリ or ブラウザ内）
- UIは1画面で完結
- エラー処理は最低限でOK

# 技術スタック指定
- フロントエンド: React + Vite
- バックエンド: Node.js + Express
  （または Next.js API Routes でも可）
- 言語: TypeScript もしくは JavaScript
- 外部API: Jira REST API v3
